filecoin技术架构分析十：filecoin源码分析之支撑包分析(2)

发表于 2019-03-07 | 更新于: 2019-03-12 | 分类于 FileCoin

我是先河系统CTO杨尉，欢迎大加关注的的Github: waynewyang，本文是filecoin技术架构分析系列文章第十章源码分析之支撑包分析(2)。

本章续上一章的支撑包介绍，主要为便于后面章节的源码理解

repo

  • 提供功能
    • 实例化fs资源或者mem资源
    • 提供读取、设置API地址方法
    • 提供存储已被校验区块的方法
    • 提供阶段密封数据存储方法
    • 提供密封完成数据存储方法
    • 提供读取配置方法
    • 提供通用数据存储方法
    • 提供交易数据存储方法
    • 提供钱包信息存储方法
    • 提供存储密钥方法
    • 提供快照配置存储方法
    • 提供版本号读取方法

proofs和sectorbuilder

  • proofs提供功能
    • 校验时空证明的方法
    • 校验密封证明的方法
    • 更细节的注释见如下代码笔者增加的注释
    • rustverifier实现具体的方法
  • sectorbuilder
    • 提供向unsealed扇区写入pieces的方法
    • 提供生成时空证明的方法
    • 提供从特定扇区读取特定pieces的方法
    • 提供密封完成通知的方法
    • 提供批量密封所有未完成的分段扇区
    • 与rust-fil-proof交互，更深入的逻辑需要参见rust

type

如下对一些主要结构进行简析

  • AttoFIL(10*-18 FIL)

    • 提供AttoFIL的算数运算方法
    • 提供AttoFIL的逻辑运算方法

  • Block

    • 区块结构
  • BlockHeight
    • 区块高度相关操作方法

  • BytesAmount (*big.Int)

    • 提供相关的算数逻辑运算

  • ChannelID(支付通道结构体)

  • 一些变量定义
    • 创建各类actor对象
  • Message相关
    • 消息结构及方法
    • filecoin网络的交易由一些列的Message组成
  • TipSet
    • 区块集合

abi

  • abi
    • 对filecoin中的各类数据定义数据类型
    • 提供abi编解码操作方法

pubsub

  • 提供功能
    • 提供订阅实例化以及订阅方法
    • 提供发布实例化以及发布方法
